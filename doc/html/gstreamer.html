3. VideoTester.gstreamer — Video processing with GStreamer bindings¶

class VideoTester.gstreamer.Gstreamer(conf, video)¶

Gstreamer library handler.

On init: Some initialization code.

Parameters:
  • conf (dictionary) – Parsed configuration file.
  • video (string) – Path to the selected video.
conf¶

Dictionary of configuration options (see VideoTester.core.Client.conf).

video¶

Path to the selected video (see VideoTester.core.Client.video).

size¶

Video size: (width, height).

files¶

Dictionary of paths to the processed video files: {'original':[<compressed>, <yuv>], 'coded':[<compressed>, <yuv>], 'received':[<compressed>, <yuv>]}.

pipeline¶

Gstreamer pipeline.

loop¶

Gstreamer loop.

url¶

Selected video URL.

encoder¶

Gstreamer encoder for the selected codec.

depay¶

Gstreamer depayloader for the selected codec.

bitrate¶

Video bitrate.

add¶

Little trick for Theora.

Note

The Theora reader (in VideoTester.video.CodedVideo) actually works with Matroska files only. This may change in future versions.

receiver()¶

Connect to the RTSP server and receive the selected video (see video).

reference()¶

Make the reference videos.

Returns:Paths to video files (see files) and video size (see size).
Return type:tuple
