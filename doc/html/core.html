2. VideoTester.core — Client and server base classes¶

class VideoTester.core.VT¶

Superclass that gathers several common functionalities shared by the client and the server.

On init: Parse the video section.

Warning

This section MUST be present in the default configuration file (see VideoTester.config.CONF) and MUST contain the same videos at the client and the server.

Raises :Bad configuration file or path.
videos¶

List of (id, name) pairs for each available video.

path¶

Path to the video directory.

run()¶

Do nothing.

Note

This method MUST be overwritten by the subclasses.

parseConf(file, section)¶

Extract a section from a configuration file.

Parameters:
  • file (string) –
  • section (string) – Path to the configuration file.
Returns:

A list of (name, value) pairs for each option in the given section.
Return type:

list of tuples
class VideoTester.core.Server¶

Bases: VideoTester.core.VT, SimpleXMLRPCServer.SimpleXMLRPCServer

VT Server class.

On init: Some initialization code.

videos¶

List of available videos (complete path).

servers¶

Dictionary of running RTSP servers.

port¶

Next RTSP port (integer). It increases each time by one.

exports¶

List of exported methods (basically, :method:`run` and :method:`stop`).

run(bitrate, framerate)¶

Run a subprocess for an RTSP server with a given bitrate and framerate (if not running) or add a client (if running).

Parameters:
  • bitrate (string or integer) – The bitrate (in kbps).
  • framerate (string or integer) – The framerate (in fps).
Returns:

The RTSP server port.
Return type:

integer
Raises OSError:

An error ocurred while running subprocess.
stop(bitrate, framerate)¶

Stop an RTSP server with a given bitrate and framerate (if no remaining clients) or remove a client (if remaining clients).

Parameters:
  • bitrate (string or integer) – The bitrate (in kbps).
  • framerate (string or integer) – The framerate (in fps).
Returns:

True.
Return type:

boolean
class VideoTester.core.Client(file, gui=False)¶

Bases: VideoTester.core.VT

VT Client class.

On init: Some initialization code.

Parameters:
  • file (string or dictionary) – Path to a configuration file (string) or parsed configuration file (dictionary).
  • gui (boolean) – True if Client is called from GUI. False otherwise.
Raises :

Bad configuration file or path.

Warning

If gui == True, file MUST be a dictionary. Otherwise, file MUST be a string.

conf¶

Dictionary of configuration options.

video¶

Path to the selected video.

run()¶
Run the client and perform all the operations:
  • Connect to the server.
  • Receive video while sniffing packets.
  • Close connection.
  • Process data and extract information.
  • Run meters.
Returns:A list of measures (see VideoTester.measures.core.Meter.measures) and the path to the temporary directory plus files prefix: <path-to-tempdir>/<prefix>.
Return type:tuple
